## Gista.js とは

Kenn Ejima (@kenn) 氏が開発中のオープンソースの Next.js スターターキットです。主に「バイブコーダー(vibe coder)」と呼ばれる非エンジニア向けに設計されており、AIコーディングアシスタントと組み合わせて本格的なWebアプリを構築・公開できることを目指しています。

AIがコードを書き、ユーザーがプロダクトをリリースする、という思想の元、ランディングページ・ブログ・ECサイトなど、ノーコードツールでは難しいカスタマイズも可能なフルスタックアプリが作れます。

### 主な特徴

- スタック：Next.js + GitHub + Vercel + Turso (DB) をベースにしており、認証・フォーム・メール・決済・自動化などの機能を備えた本格的なSaaSが無料枠で構築・デプロイできます。
- ターゲット：「ターミナルって何？」というレベルの人でも使えるよう、環境構築の手順から丁寧に説明されており、手順を追うだけでDB・ログイン・メールの理解が自然に深まる設計になっています。
- AIフレンドリー：AIコーディングアシスタント（Claude、Codex、Copilotなど）が正しくコードを生成できるよう、明確なパターンと一貫した規約が整備されています。
- エコシステム：OSSコア・デモアプリ・CLI・有料アドオンを誰でも販売できるエコシステムも構想されており、単なるスターターを超えたプラットフォームを目指しています。

### 個人的に気になったところ

教育的側面：Gista.jsが目指しているのは「作りながら学ぶ」という体験を一つのツールに統合することで、その意味では「開発ツール」と「教育プロダクト」の中間にある新しいカテゴリに属する。作者の Kenn Ejima 氏自身も「日エンジニアのバイブコーダー向けという教育面の大きい製品」と表現している。

Never delegate understanding：「理解をAIに丸投げするな」という言葉がサイトにある。AIがキーストロークを担う時代になると、コードを書く能力は安くなり、読む能力こそが生存に必要なスキルになる、という思想が根底にある。

### 感想
このスターターキットの「AIにコードを書かせながら、コードを理解することも促す」という思想は自分がアプリ制作をAIに丸投げせずに会話のラリーをしながらあくまで実装は理解していくというスタンスに似ていると感じた。
しかし、このアプローチは現時点でエンジニアでない者がエンジニアリングを学習していこうとするアプローチとしてはデファクトスタンダードになっているのではないかと思うので当たり前のことかもしれないが、Gista.js はスターターキットと分類されるようにその時に必要なものをまとめてくれているということなのかなと落とし込んだ。
面白そうなので、余裕ができたら触っていきたい。
