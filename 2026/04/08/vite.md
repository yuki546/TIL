# TIL: Vite とは何か

## 概要

Vite（ヴィート）は、フランス語で「速い」を意味するフロントエンド向けのビルドツール。
Evan You（Vue.js の作者）が開発し、開発サーバーの起動・HMR（ホットモジュールリプレース）の
高速化を主な目的として設計されている。

従来の Webpack などのバンドラーは、起動時にすべてのモジュールをバンドルしてから
サーバーを立ち上げるため、プロジェクトが大きくなるほど起動が遅くなる問題があった。
Vite はブラウザネイティブの ES Modules（ESM）を活用することで、この問題を解決している。

---

## メリット

- **起動が速い**：事前バンドルが不要なため、開発サーバーの起動がほぼ一瞬
- **HMR が高速**：変更したモジュールだけを差し替えるので、コード変更の反映がリアルタイムに近い
- **設定が少ない**：ゼロコンフィグに近く、React / Vue / Svelte など主要フレームワークのテンプレートを公式サポート
- **本番ビルドは Rollup**：最適化された成果物を生成できる

---

## デメリット

- **CommonJS との相性**：ESM ベースのため、CJS 形式のパッケージは内部で変換処理が走ることがある
- **開発・本番の差異**：開発時は ESM のままサーブ、本番は Rollup でバンドルするため、稀に挙動の差が出ることがある
- **エコシステムはまだ Webpack より小さい**：プラグインの数や事例は Webpack に比べると少ない（ただし急速に拡大中）

---

## プロジェクト作成コマンド

```bash
# 対話形式でプロジェクトを作成（推奨）
npm create vite@latest

# プロジェクト名とテンプレートを直接指定する場合
npm create vite@latest my-app -- --template react-ts
```

テンプレート一覧の例：

| テンプレート名 | 内容 |
|---|---|
| `vanilla` | フレームワークなし（JS） |
| `react` | React（JS） |
| `react-ts` | React（TypeScript） |
| `vue` | Vue 3（JS） |
| `vue-ts` | Vue 3（TypeScript） |

プロジェクト作成後の流れ：

```bash
cd my-app
npm install
npm run dev
```
